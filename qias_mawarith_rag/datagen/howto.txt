## Phase 4: Training Options

### Option A: SFT Training (Supervised Fine-Tuning)
Standard training on input-output pairs:

```bash
python3 training/train_unsloth.py
# Options:
python3 training/train_unsloth.py --epochs 3 --batch-size 2
```

### Option B: GRPO Training (Recommended) 🎯
Reinforcement Learning with custom reward functions:

```bash
python3 training/train_grpo.py
# Options:
python3 training/train_grpo.py --max-steps 500 --num-generations 4
```

**GRPO Reward Functions:**
1. **Format Compliance** (+2.5 max): Checks <think> and <answer> tags
2. **Thinking Quality** (+5.0 max): Analyzes Fiqh keywords in reasoning
   - الحجب (blocking), الفرض (fixed share), التعصيب (residuary)
   - العول (increase), الرد (return)
3. **JSON Validity** (+4.0 max): Verifies answer structure
4. **Calculator Verification** (+5.0 max): Uses symbolic calculator

## Training Data Format

```json
{
  "messages": [
    {"role": "system", "content": "أنت خبير في علم المواريث..."},
    {"role": "user", "content": "توفي شخص وخلّف: زوجة وابن..."},
    {"role": "assistant", "content": "<think>نبدأ بتحليل...</think>\n<answer>{...}</answer>"}
  ]
}
```

---

## Qwen3-2507 Thinking Best Practices

From Unsloth docs:
- **temperature**: 0.6 (not 0.7 like Instruct)
- **top_p**: 0.95
- **top_k**: 20
- **min_p**: 0.0
- **max tokens**: 32,768 for most queries

The default chat template adds <think> automatically, so you may see only a closing </think> tag in the output.
